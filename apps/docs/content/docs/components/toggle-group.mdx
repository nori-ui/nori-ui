## At a glance

- Clusters several [`<Toggle>`](/docs/components/toggle)-style buttons with shared selection semantics.
- `type="multiple"` — any-of-N (formatting marks). `value` is `string[]`.
- `type="single"` — one-of-N, optionally none (text alignment). `value` is `string | undefined`. Re-clicking the active item clears the selection.
- Roving-tabindex keyboard nav follows the WAI-ARIA radiogroup pattern: `Arrow` keys move focus, `Home` / `End` jump to the ends, `Space` / `Enter` toggles the focused item.
- ARIA: every item exposes `role="button"` + `aria-pressed`. The group is `role="group"` (multiple) or `role="radiogroup"` (single).
- Default-variant grouped items share borders so the cluster reads as one segmented chip — first item rounds the left, last rounds the right, middle is square.

<Callout title="ToggleGroup vs SegmentedControl vs Radio.Group">
    - **ToggleGroup multiple** — toolbar-style. Several can be active at once (Bold + Italic).
    - **ToggleGroup single** — like a SegmentedControl, but the group can be cleared by re-clicking the active item.
    - **[SegmentedControl](/docs/components/segmented-control)** — exactly one is always active. Use this when "no selection" isn't a valid state.
    - **[Radio.Group](/docs/components/radio)** — the form-y radio look. Use this in forms where the user is reviewing choices and submitting later.
</Callout>

## Multiple selection (formatting toolbar)

`type="multiple"` — `value` is `string[]`. Clicking adds or removes an item from the array.

<Preview name="toggle-group-multiple" />

## Single selection (alignment)

`type="single"` — `value` is `string | undefined`. Clicking sets it; clicking the active item again clears it.
