## At a glance

- Compose: `Popover`, `PopoverTrigger`, `PopoverContent`. Trigger uses `asChild` by default â€” wrap any element (Button, Link, custom Pressable) and it becomes the activator.
- Cross-platform: web uses `position: fixed` with a measured trigger rect so it escapes any ancestor `overflow: hidden`. Native uses RN `<Modal>` with a transparent backdrop and tap-outside-to-close.
- Non-modal: focus is NOT trapped â€” the user can tab back out as normal. Dismisses on outside click and Escape.
- Accessibility: trigger gets `aria-haspopup="dialog"` + `aria-expanded`; content gets `role="dialog"` (no `aria-modal`).

<Callout type="info" title="Popover vs Tooltip vs Dialog">
    - **Popover** â€” anchored, non-modal, can hold interactive content (forms, buttons, color pickers). Dismisses on outside click + Escape.
    - **Tooltip** â€” anchored, non-interactive, short hover-triggered hints (one or two words to a sentence). Use Tooltip when there's nothing to click inside.
    - **Dialog** â€” modal, centered, traps focus, locks scroll. Use Dialog when the user must respond before continuing.
</Callout>

## Anatomy

| Subcomponent | Role |
|---|---|
| `Popover` | Root â€” owns open state (controlled or uncontrolled). |
| `PopoverTrigger` | Element that toggles the popover. `asChild` by default. |
| `PopoverContent` | The floating surface. Renders only while open. |

## Positioning

`PopoverContent` accepts:

- `side` â€” `'top' | 'right' | 'bottom' | 'left'` (default `'bottom'`). Which edge of the trigger to anchor on.
- `align` â€” `'start' | 'center' | 'end'` (default `'center'`). Alignment along the chosen edge.

There is a 4-pixel gap between the trigger and the content so the two don't kiss. The minimum width is 200 px so help text never wraps awkwardly.

## Controlled vs uncontrolled

```tsx
// Uncontrolled â€” Popover owns its open state.
<Popover>
    <Popover.Trigger><Button>Open</Button></Popover.Trigger>
    <Popover.Content>...</Popover.Content>
</Popover>

// Controlled â€” you own the state, useful for closing from inside the content
// after a save, or for syncing open with route state.
const [open, setOpen] = useState(false);

<Popover open={open} onOpenChange={setOpen}>
    <Popover.Trigger><Button>Open</Button></Popover.Trigger>
    <Popover.Content>...</Popover.Content>
</Popover>
```
