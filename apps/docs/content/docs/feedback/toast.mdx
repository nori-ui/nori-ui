## At a glance

- Wrap your app (or the section that needs toasts) in `<ToastProvider>`. Then call `useToast().toast(...)` from any descendant.
- Auto-dismisses after `duration` ms (default 5000). Pass `duration: Infinity` to keep the toast open until dismissed manually.
- Each toast supports `tone` (info / success / warning / danger), `description`, and an optional `action` button (typically "Undo" or "View").
- Viewport renders bottom-right by default; toasts stack newest at the bottom.

## API

`toast({ title, description?, tone?, duration?, action? })` returns the toast id. `dismiss(id)` removes a single toast; `dismissAll()` clears the queue.
