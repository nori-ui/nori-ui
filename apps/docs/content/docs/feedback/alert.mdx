## At a glance

- **Tones:** `info`, `success`, `warning`, `danger`. Each picks a default icon from the semantic icon set.
- Use Alert for messages the reader should *notice* but doesn't need to act on right away. Reach for [Dialog](/docs/overlays/dialog) when the user must acknowledge before continuing.
- `role="alert"` is set automatically so screen readers announce the message.
- Pass `onDismiss` to render a close button. The Alert never tracks dismissed state itself — the parent decides what to render after.
